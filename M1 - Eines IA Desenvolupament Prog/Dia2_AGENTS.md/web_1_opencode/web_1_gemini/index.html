Mòdul 01

Introducció a la Programació i Algorismes

La programació és la resolució de problemes mitjançant la lògica codificada. Abans d'escriure qualsevol línia de codi en un llenguatge de programació, cal entendre com definir una seqüència estructurada de passos.

El Concepte d'Algorisme

Un algorisme és un conjunt ordenat, definit i finit d'instruccions o regles que permeten realitzar una activitat, resoldre un problema o prendre una decisió.

Els algorismes tenen tres característiques fonamentals:

  • Precisió: Cada pas ha d'estar clarament definit sense ambigüitats.
  • Determinisme: Donades les mateixes entrades, el resultat sempre ha de ser el mateix.
  • Finitud: L'algorisme ha d'acabar en algun moment, després de realitzar un nombre de passos finits.

Exemple d'Algorisme en Pseudocodi

Disseny d'un flux per determinar si un alumne aprova o suspèn una assignatura:

Algorisme DeterminarAprovat
    Llegir nota_examen
    Llegir nota_practiques
    
    mitjana = (nota_examen + nota_practiques) / 2
    
    Si mitjana >= 5 Llavors
        Escriure "Aprovat amb un " + mitjana
    Sino
        Escriure "Suspès amb un " + mitjana
    FiSi
FiAlgorisme
Mòdul 02

Variables, Constants i Tipus de Dades

Per a poder treballar amb informació, els programes necessiten emmagatzemar valors temporals en la memòria de l'ordinador utilitzant identificadors anomenats variables i constants.

Declaració de Dades

En el desenvolupament modern (JavaScript), disposem de tres maneres clau de declarar dades, tot i que avui dia se'n desaconsella una:

  • let: Declara una variable d'àmbit de bloc que pot canviar el seu valor al llarg del programa (reassignació).
  • const: Declara una constant de només lectura, el valor de la qual no es pot reassignar una vegada definit inicialment.
  • var: La forma clàssica d'esdevenir variables, que té àmbit de funció i és propensa a errors a causa del fenomen del hoisting. Evitar-ne l'ús!

Tipus de Dades Primitius

El tipus de dada defineix el format, el comportament i la quantitat d'espai de memòria dedicat a emmagatzemar la informació. Tipus primitius clau:

Tipus Descripció Exemple
Number Valors numèrics (enters o decimals) 42 o 19.99
String Cadenes de text delimitades per cometes "Hola món"
Boolean Estats lògics bivalents true o false
Null Absència intencionada de valor null
Undefined Variables no definides o sense valor inicial undefined
Symbol / BigInt Valors únics o enters de precisió arbitrària 123n

Exemple Pràctic: Declaració i Tipat Dinàmic

// Constants i Variables
const PI = 3.14159;
let radiCircumferencia = 12.5;
let area;

// Càlcul matemàtic
area = PI * (radiCircumferencia ** 2);

// Demostració de reassignació i tipat dinàmic
let usuariActiu = "Arnau"; // Tipus String
console.log(typeof usuariActiu); // Imprimeix "string"

usuariActiu = true; // Ara és de tipus Boolean
console.log(typeof usuariActiu); // Imprimeix "boolean"
Mòdul 03

Operadors i Expressions

Els operadors són símbols especials que permeten realitzar operacions de càlcul, comparació o lògica amb un o més operands (valors o variables).

Classificació d'Operadors

Per a dominar la lògica de programació, cal entendre com funcionen i s'avaluen els següents operadors bàsics:

  • Aritmètics: Permeten fer càlculs matemàtics (+, -, *, /, % mòdul, ** exponenciació).
  • D'Assignació: Assignen un valor a una variable, sovint amb operacions acumulatives (=, +=, -=, *=, /=).
  • De Comparació: Avaluen la relació entre dos valors i retornen un booleà:
    • Comparació estricta (recomanada): === (igual valor i tipus), !== (diferent valor o tipus).
    • Comparació feble (a evitar): ==, !=.
    • Relacionals: >, <, >=, <=.
  • Lògics: Permeten construir condicions complexes (&& AND lògic, || OR lògic, ! NOT lògic).

Exemple de Lògica de Condicions

Avaluació d'accés a una plataforma escolar basat en el pagament de la matrícula o una beca, combinat amb l'edat de l'usuari:

const edatAlumne = 17;
const teMatriculaPagada = true;
const teBecaAprovada = false;

// Lògica combinada
const potAccedirAlCurs = (teMatriculaPagada || teBecaAprovada) && edatAlumne >= 16;

console.log("Accés concedit:", potAccedirAlCurs); // true

// Operació aritmètica combinada amb mòdul (parell o senar)
const esEdatParell = (edatAlumne % 2) === 0;
console.log("Edat parella?", esEdatParell); // false
Mòdul 04

Estructures Condicionals i Control de Flux

El control de flux permet que l'execució d'un bloc de codi sigui dinàmica, seleccionant camins alternatius basant-se en l'avaluació de condicions booleanes.

La Bifurcació Lògica

Existeixen tres mètodes principals per a redirigir el flux d'execució segons les dades del programa:

Estructura if / else if / else
Avalua condicions seqüencialment. Si una és vertadera, n'executa el bloc intern i ignora les posteriors.
Estructura switch
Molt útil per a simplificar múltiples condicions d'igualtat sobre una sola variable d'entrada. Recomanat quan es treballa amb llistes tancades d'estats o menús.
Operador Ternari (condició ? expr1 : expr2)
Sintaxi condensada d'una estructura simple d'elecció. Ideal per a realitzar assignacions ràpides i directes.

Exemple d'Estructures en Acció

// 1. Exemple de Switch en Català
const tipusSubscripcio = "premium";
let preuMensual;

switch (tipusSubscripcio) {
  case "basic":
    preuMensual = 9.99;
    break;
  case "standard":
    preuMensual = 14.99;
    break;
  case "premium":
    preuMensual = 19.99;
    break;
  default:
    preuMensual = 0.00;
}

// 2. Exemple d'Operador Ternari
const estatPlataforma = preuMensual > 0 ? "De pagament" : "Gratuïta";
console.log(`El servei és: ${estatPlataforma}`);
Mòdul 05

Bucles i Iteració

La repetició és un dels grans superpoders dels ordinadors. Un bucle o estructura repetitiva executa de forma reiterada un bloc d'instruccions fins que es trenca o deixa de complir-se una condició determinada.

Estructures Iteratives Fonamentals

Per implementar la repetició de tasques utilitzem diferents sintaxis segons si coneixem per endavant la quantitat de repeticions:

  • Bucle for: Es fa servir quan el nombre de repeticions o iteracions és conegut abans d'iniciar el bucle. Disposa de tres parts: inicialització, condició de parada i increment.
  • Bucle while: Executa el codi tantes vegades com la condició continuï sent vertadera. S'avalua la condició abans d'entrar al bloc.
  • Bucle do...while: Similar al while, amb la diferència que la condició s'avalua després de cada execució, garantint que el bloc s'executi com a mínim un cop.

Codificació dels Diferents Bucles

// Exemple de bucle FOR per comptar
for (let i = 1; i <= 5; i++) {
  console.log(`Iteració número ${i}`);
}

// Exemple de bucle WHILE per cercar un llindar
let energia = 100;
let intents = 0;

while (energia > 15) {
  energia -= 25; // reduïm energia gradualment
  intents++;
  console.log(`Intent ${intents}: Energia restant: ${energia}`);
}

// Exemple de DO-WHILE (garanteix una execució)
let clauValida = false;
do {
  // Executa el codi com a mínim un cop
  console.log("Comprovant seguretat del sistema...");
  clauValida = true; // Forcem sortida
} while (!clauValida);
Mòdul 06

Funcions i Àmbits (Scope)

Les funcions són blocs de codi autònoms dissenyats per fer una tasca específica. Ens permeten encapsular codi per evitar duplicitats, facilitar proves i garantir la reutilització.

Sintaxi i Àmbits de Variable

Podem declarar funcions de diverses maneres, incloent-hi la sintaxi moderna de funcions fletxa, molt utilitzades en JavaScript modern:

  • Funcions Declaratives: Tenen nom i se'ls aplica l'hissat (poden cridar-se abans de la seva definició escrita).
  • Funcions d'Expressió o Fletxa (Arrow): No tenen nom (són anònimes), es desen en variables i tenen una sintaxi compacta ideal per a callbacks.
  • Àmbit (Scope): Determina l'accessibilitat de les variables:
    • Global: Accessible des de tot el programa.
    • Local o de Funció: Declarades dins d'una funció, inaccessibles des de fora.
    • De Bloc: Variables declarades amb let o const dins d'unes claus `{}`.

Exemple de Funcions i Àmbits

// Funció Declarativa Tradicional
function calcularDescompte(preu, percentatge = 10) {
  const estalvi = (preu * percentatge) / 100;
  return preu - estalvi;
}

// Funció Fletxa Moderna (un sol paràmetre, retorn implícit)
const calcularIva = preu => preu * 1.21;

// Ús pràctic i demostració de Scope
const preuProducte = 100;
const preuAmbIva = calcularIva(preuProducte);
const preuFinal = calcularDescompte(preuAmbIva, 15);

console.log(`Preu Final: ${preuFinal.toFixed(2)}€`);
// console.log(estalvi); // ERROR: estalvi només existeix a calcularDescompte
Mòdul 07

Arrays i Objectes Literals

Per a modelar la realitat, els tipus primitius són insuficients. Necessitem estructures capaces d'emmagatzemar múltiples dades agrupades, ordenades o etiquetades.

Arrays (Llistes Ordenades)

Un array és una col·lecció indexada d'elements ordenada. Comença sempre pel seu índex zero (0).

A JavaScript, els arrays són altament dinàmics i ofereixen una gran varietat de mètodes natius que simplifiquen enormement la manipulació de la informació:

  • push(elem): Afegeix un element al final de l'array.
  • pop(): Elimina i retorna l'últim element.
  • shift() / unshift(elem): Manipulen els elements pel davant.
  • length: Propietat que indica la mida o quantitat d'elements.

Objectes Literals (Estructures de Clau-Valor)

Un objecte és una entitat independent que conté propietats i mètodes. Permet associar claus (etiquetes de text) amb valors de qualsevol tipus de dada.

const alumne = {
  nom: "Marta",
  cognom: "Gómez",
  edat: 21,
  curs: "Eines de Desenvolupament IA",
  saludar: function() {
    return `Hola, sóc la ${this.nom}!`;
  }
};

console.log(alumne.nom); // Accés per punt: "Marta"
console.log(alumne.saludar()); // Execució de mètode

Exemple Combinat: Array d'Objectes amb Iteració Complexa

// Llista de tasques d'aprenentatge
const llistaTasques = [
  { id: 1, descripcio: "Repassar condicional switch", finalitzada: true },
  { id: 2, descripcio: "Practicar mètodes d'arrays", finalitzada: false },
  { id: 3, descripcio: "Configurar entorn Git", finalitzada: true }
];

// Iteració sobre la llista per comptar tasques pendents
let tasquesPendents = 0;

for (let i = 0; i < llistaTasques.length; i++) {
  const tasca = llistaTasques[i];
  if (!tasca.finalitzada) {
    tasquesPendents++;
    console.log(`Pendent: ${tasca.descripcio}`);
  }
}

console.log(`Total de tasques pendents d'enllestir: ${tasquesPendents}`);
Mòdul 08

Programació Orientada a Objectes (POO)

La POO és un paradigma de programació que utilitza objectes per representar estructures de dades del món real, facilitant l'organització i manteniment del codi a gran escala.

Els Quatre Pilars de la POO

El disseny orientat a objectes es fonamenta en conceptes sòlids que permeten estructurar aplicacions robustes:

Classes i Instàncies
La classe és el motlle (o model genèric), i l'objecte o instància és l'element concret creat a partir d'aquest motlle mitjançant la paraula clau new.
El Constructor
Mètode especial d'inicialització que s'executa automàticament en instanciar l'objecte, utilitzat per rebre els valors inicials i configurar-los.
Herència (Inheritance)
Mecanisme que permet que una nova classe (filla) hereti els atributs i mètodes d'una classe existent (pare), promovent la reutilització estricta del codi.
Encapsulament i Polimorfisme
Protecció interna de les dades per evitar modificacions directes no controlades, i capacitat d'executar mètodes amb el mateix nom però comportaments diferents.

Codificació de Classes i Herència

// Classe Pare (Superclasse)
class Usuari {
  constructor(nom, email) {
    this.nom = nom;
    this.email = email;
    this.rol = "Estudiant";
  }

  descriurePerfil() {
    return `${this.nom} (${this.rol}) - Correu: ${this.email}`;
  }
}

// Classe Filla (Subclasse)
class Professor extends Usuari {
  constructor(nom, email, especialitat) {
    super(nom, email); // Crida al constructor de la superclasse
    this.rol = "Professor";
    this.especialitat = especialitat;
  }

  // Polimorfisme: Sobreestrivim el mètode original
  descriurePerfil() {
    return `${super.descriurePerfil()} | Especialitat: ${this.especialitat}`;
  }
}

// Instanciació d'objectes concrets
const professorCurs = new Professor("Marc Estela", "m.estela@escolatec.cat", "Intel·ligència Artificial");
console.log(professorCurs.descriurePerfil());
