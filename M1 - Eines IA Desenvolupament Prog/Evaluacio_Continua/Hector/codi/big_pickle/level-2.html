Object-Oriented Programming

OOP organises code around objects rather than functions and logic. An object bundles data (attributes) and behaviour (methods) into a single unit.

Four Pillars

  • Encapsulation — hiding internal state and requiring all interaction through methods
  • Inheritance — creating new classes based on existing ones to reuse code
  • Polymorphism — using a single interface to represent different underlying forms
  • Abstraction — exposing only essential features and hiding implementation details

Example

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        pass

class Dog(Animal):
    def speak(self):
        return f"{self.name} says Woof!"

class Cat(Animal):
    def speak(self):
        return f"{self.name} says Meow!"

Data Structures

Data structures organise and store data efficiently. Choosing the right structure is key to writing fast, scalable programs.

Arrays / Lists

Ordered collections of elements accessible by index. Good for sequential access and iteration.

numbers = [3, 7, 1, 9, 4]
numbers.append(2)
first = numbers[0]     # 3
sorted_nums = sorted(numbers)

Stacks and Queues

  • Stack — Last-In-First-Out (LIFO). Like a pile of plates.
  • Queue — First-In-First-Out (FIFO). Like a line of people.

Trees

A hierarchical structure with a root node and child nodes. Binary trees are widely used in searching and sorting.

Rule of thumb: Arrays for fast index access, linked lists for frequent insertions/deletions, hash tables for fast lookups by key.

Algorithms

An algorithm is a step-by-step procedure for solving a problem. The study of algorithms focuses on correctness and efficiency.

Sorting

# Bubble sort — O(n²)
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

Searching

  • Linear search — check each element in order. O(n).
  • Binary search — repeatedly halve a sorted array. O(log n).

Complexity (Big O)

Big O notation describes how runtime or memory grows as input size increases. Common classes:

  • O(1) — constant time (hash lookup)
  • O(log n) — logarithmic (binary search)
  • O(n) — linear (single loop)
  • O(n log n) — log-linear (merge sort)
  • O(n²) — quadratic (nested loops)

Memory Management

Understanding how memory works helps write efficient and bug-free code.

Stack vs Heap

  • Stack — stores local variables and function call frames. Fast, limited size, automatically managed.
  • Heap — stores dynamically allocated data. Larger, slower, requires explicit management or a garbage collector.

Common Issues

  • Memory leak — memory that is no longer needed is not freed
  • Dangling pointer — a reference to memory that has been freed
  • Buffer overflow — writing beyond the bounds of allocated memory
Languages with garbage collection (Java, Python, Go) handle heap cleanup automatically, but you should still avoid holding references longer than necessary.

Error Handling

Errors are inevitable. Robust programs anticipate and handle them gracefully rather than crashing.

Exceptions

Exceptions are events that disrupt normal program flow. Most modern languages provide structured exception handling.

Example (Python)

try:
    result = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero!")
except TypeError as e:
    print(f"Type error: {e}")
finally:
    print("This always runs.")

Best Practices

  • Catch specific exceptions, not broad ones
  • Fail fast — detect errors early
  • Log errors with enough context to diagnose them
  • Never expose internal error details to end users
