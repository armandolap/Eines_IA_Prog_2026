Level 1: Foundational Programming

Comprehensive guide to variables, typing systems, conditionals, loops, functions, scopes, and standard computer I/O.

1. Variables & Data Types

At its core, programming is about managing and manipulating data. A variable is a symbolic name given to a specific location in the computer's memory that stores a value.

Primitive Data Types

Most general-purpose programming languages offer a set of fundamental or "primitive" types. These are built directly into the language syntax and form the bedrock of more complex structures:

Type Description Common Sizes Examples
Integer Whole numbers (positive, negative, or zero) without decimals. 32-bit (standard), 64-bit (long) -5, 0, 42
Float / Double Real numbers that contain fractional or decimal parts. 32-bit (float), 64-bit (double) 3.14159, -0.001
Boolean A logical value representing truth or falsehood. 1 byte (typically) true, false
Character A single alphanumeric symbol or control code. 1 byte (ASCII), 2-4 bytes (Unicode) 'A', '?', '\n'
String A sequence of characters (often treated as an object or primitive depending on the language). Variable "Hello, World!"

Static vs. Dynamic Typing

Languages categorize how variables are declared and bound to data types into two main categories:

  • Static Typing: Variable types are declared explicitly at compile-time and cannot change. (Examples: C++, Java, Rust).
  • Dynamic Typing: Variable types are determined automatically at runtime based on the value assigned to them. (Examples: Python, JavaScript, Ruby).
Statically Typed (C++) Example code 
// Variables must declare their data type explicitly
int age = 25;
double temperature = 98.6;
std::string username = "alice_dev";

// This would trigger a compilation error:
// age = "twenty-five";
Dynamically Typed (Python) Example code 
# Types are inferred at runtime
age = 25
temperature = 98.6
username = "alice_dev"

# Re-assigning to a different type is completely legal
age = "twenty-five"
💡 Type Safety and Casting

Even in statically-typed languages, you can convert one data type into another. This is known as type casting. It can be implicit (done automatically, e.g., converting an integer to a float during addition) or explicit (forced by the programmer).

2. Control Flow & Loops

By default, a computer executes code sequentially from top to bottom. Control flow structures allow us to break this linear path, executing certain blocks conditionally or repeating blocks of code multiple times.

Conditionals (If-Else)

Conditionals evaluate logical expressions to determine which branch of code to execute. They rely on comparison operators (==, !=, <, >, <=, >=) and logical operators (AND, OR, NOT).

Python Conditional Control flow example 
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
else:
    grade = "C"

Loops (For & While)

Loops enable repetitive execution of instructions. There are two primary varieties:

  • For Loops: Typically used when the number of iterations is known in advance (e.g., iterating over a range of numbers or collection items).
  • While Loops: Used when the number of iterations depends on a boolean condition that is evaluated before each loop iteration.
JavaScript Loops Repetitive iteration 
// For loop: repeats 5 times
for (let i = 0; i < 5; i++) {
    console.log("Iteration: " + i);
}

// While loop: runs until condition becomes false
let balance = 100;
while (balance > 10) {
    balance -= 20;
}
⚠️ The Infinite Loop Hazard

An infinite loop occurs when the loop termination condition is never met. This can consume 100% of the CPU allocation for the process, causing the program (and sometimes the host environment) to freeze. Ensure variables within your while loop condition are updated correctly in the loop body!

3. Functions & Scope

A function (also known as a procedure or method) is a self-contained block of statements designed to perform a specific task. Functions allow code to be written once and reused multiple times, promoting the DRY (Don't Repeat Yourself) principle.

Function Anatomy

  • Signature: The function's name and its parameter list.
  • Parameters: Placeholder variables defined in the function declaration that accept values (arguments) when the function is called.
  • Return Value: The output value sent back to the caller once the function completes. If no value is returned, it is often called a void function.
JavaScript Function Declaration and call 
// Function definition with two parameters
function calculateTotal(price, taxRate) {
    let tax = price * taxRate;
    return price + tax; // returns a value to the caller
}

// Function invocation with actual arguments
let finalPrice = calculateTotal(50.0, 0.08);
console.log(finalPrice); // Outputs: 54

Variable Scope

Scope determines the visibility and lifetime of variables in your code. Generally, scopes are divided into:

  1. Global Scope: Variables declared outside of any block or function. They are visible throughout the entire program's lifecycle.
  2. Local Scope: Variables declared inside a specific function or code block. They are created when the function starts and destroyed when the function finishes, meaning they cannot be accessed from outside that function.
Python Scope Example Demonstrating variable boundaries 
global_msg = "I am everywhere" # Global Scope

def my_function():
    local_msg = "I am secret" # Local Scope
    print(global_msg)         # Valid: can read global
    print(local_msg)          # Valid: local variable

my_function()
# print(local_msg)            # NameError: local_msg is not defined!

4. Input & Output (I/O)

An isolated program is of little use. Programs must communicate with the outside world by receiving input and returning output.

Standard Streams

In command-line environments, computers handle basic interactive communication using three standard operating system streams:

  • Standard Input (stdin): Typically feeds keystrokes from the user's keyboard to the program.
  • Standard Output (stdout): The default stream where the program outputs text (typically rendered to the user's terminal console).
  • Standard Error (stderr): A separate stream dedicated to outputting diagnostic or error reports, separate from standard output.

File I/O

To preserve data permanently, programs read from and write to storage devices using File I/O. This involves interacting with the operating system filesystem to open files, buffer data, and securely close the file handlers.

Python File I/O Reading and writing data 
# Writing data to a local text file
with open("output.txt", "w") as file:
    file.write("This is stored in storage!\n")

# Reading data back from the file
with open("output.txt", "r") as file:
    content = file.read()
    print(content)
💡 Why We Must Close Files

When a program opens a file, the operating system assigns a file descriptor to keep track of it. These are limited system resources. If your program opens many files without closing them, it may hit OS resource exhaustion limits, leading to file access errors or system crashes. Modern construct blocks like Python's with statement handle automatic closing safely.
